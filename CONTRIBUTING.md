## 3. How to Submit Changes

### 3.1 Fork the Repository
Standard open-source flow:
1. Fork the repo  
2. Create a feature branch  
   ```
   git checkout -b feature/your-topic
   ```
3. Make your edits  
4. Commit with a clear message:  
   ```
   git commit -m "Add: dignity metric refinement"
   ```
5. Push and create a Pull Request (PR)

---

## 4. Pull Request Guidelines

Your PR should include:
- **What you changed**  
- **Why you changed it**  
- **How it aligns with ORI principles**  
- If AI assisted any part of the writing/coding:  
  include a note such as  
  > ‚ÄúAI-assisted drafting; human reviewed.‚Äù

Before approval, maintainers will check:
- Ethical consistency  
- Technical soundness  
- Security safety  
- Metadata header presence (when needed)  
- Schema validity (for JSON/YAML specs)

---

```

spec_version: "1.0"
```

Files that **should NOT** include metadata:
- `.json` schemas  
- Code files (e.g., `.py`, `.js`)  
- Binary data

## 6. Code Style & Safety

### 6.1 Code style
- Prefer clarity > cleverness  
- Use descriptive variable names  
- Avoid unnecessary dependencies  
- Keep functions short and testable  

### 6.2 Safety requirements
Contributed code **must not**:
- Hide logic or create untraceable flows  
- Embed persuasion algorithms  
- Manipulate user emotions  
- Introduce dominance-seeking behavior

## 7. AI-Assisted Contributions
AI-generated text or code is allowed *only if*:
- It is clearly marked  
- A human maintainer reviews it  
- It does not introduce hallucinated facts  
- It does not violate safety or dignity principles  

Example notice:
> ‚ÄúDrafted with AI; reviewed and corrected by contributor.‚Äù
